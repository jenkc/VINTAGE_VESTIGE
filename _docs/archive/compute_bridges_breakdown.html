Inline Classification

Every bridge is classified at insertion time via classify_bridge(). Six dimensions stored on each row.

bridge_type
Which pass: similarity opposition structural visual_echo
temporal_type
Time distance: echo 80+ yr · transmission 40-79 yr · continuation 20-39 yr same era · contemporary <20 yr
crossing_type
Context: cross_category_culture cross_culture cross_category same_context
connection_mode
contrast opposite axis poles (struct ≥ 0.4) · resonance text_sim ≥ 0.80 + transmission/echo · affinity default
primary_axis
Design dimension: volume ornament exposure gender register occasion
contrast_pair
e.g. Exaggerated Volume <-> Column Minimalism · null for non-contrast

Connection Mode Priority

1. CONTRAST Is bridge_type == 'opposition'? OR are products on opposite axis poles with structural ≥ 0.4? → connection_mode = 'contrast', axis from opposition pair 2. RESONANCE text_sim ≥ 0.80 AND temporal_type in (transmission, echo)? → connection_mode = 'resonance', axes from shared attributes 3. AFFINITY Everything else → connection_mode = 'affinity', axes from shared attributes

Opposition Pass Scoring

Pass 2 candidates are ranked by a composite that rewards bridges crossing the most boundaries:

sort_score = 0.40 × cross_cultural_overlap (Jaccard across 3 fields, normalized) + 0.35 × temporal_distance (year gap / 200, capped at 1.0) + 0.25 × structural_score (Fashionpedia field overlap)

Top 150 per axis pair kept, with participation cap of 8 per product. When a pole side has >200 products, subsampled to 200 by cross-cultural field richness.

Too-Similar Gate

Prevents near-duplicate pairs from becoming bridges. A high score on one dimension is fine (visually similar but different description = interesting). Both high = same thing.
